# План работ

## Этап 1. Импорт правил и карт

Цель: получить воспроизводимый пайплайн, который из локальных PDF создает изображения отдельных карт, JSON-базу карт и формальное описание правил.

1. Проанализировать входные файлы в корне проекта.
   - Правила: PDF с извлекаемым текстом и/или названием, похожим на rulebook.
   - Карты: PDF-листы без текстового слоя, требующие рендера, сегментации и OCR/review.
2. Создать структуру проекта.
   - `data/processed/pages/` для отрендеренных и предобработанных страниц.
   - `data/processed/card_images/` для отдельных карт.
   - `data/processed/ocr/` для промежуточного OCR.
   - `data/review/` для ручной проверки.
   - `docs/` для решений, модели правил и отчетов.
3. Реализовать `python -m src.importers.run_import`.
   - Автоматически находить PDF.
   - Рендерить страницы через PyMuPDF.
   - Обрезать белые поля и повышать контраст через Pillow.
   - Сегментировать листы на отдельные карты по проекциям непустых пикселей.
   - Поддержать ручные overrides через `data/review/segmentation_overrides.json`.
   - Сохранять координаты карт на странице.
4. Реализовать первичную JSON-схему карт.
   - Не выдумывать данные: если OCR недоступен или не уверен, поля остаются пустыми/null.
   - Все такие записи получают `ocr.needs_review: true`.
   - Сохранять `data/processed/cards.json` и `data/review/cards_needs_review.json`.
5. Извлечь текст правил из rulebook PDF.
   - Сохранить сырой текст в `data/processed/rules_text.txt`.
   - Создать `data/processed/rules_summary.json`.
   - Создать `docs/rules_model.md` с формальным описанием по данным из правил и явными `needs_review` для спорных мест.
6. Добавить тесты.
   - Проверить валидность структуры `cards.json`.
   - Проверить обязательные поля карты.
   - Проверить, что сомнительные записи попадают в review-файл.
7. Обновить документацию.
   - `README.md` с командами запуска.
   - `docs/import_pipeline.md`.
   - `docs/decisions.md`.
   - `docs/progress_stage_1.md`.

## Ограничения текущего прохода

- На машине найден Python и pytest, но не найден системный Tesseract.
- OCR будет опциональным: если движок OCR отсутствует, пайплайн все равно создает изображения и JSON-записи, но помечает карты как требующие ручной проверки.
- Этапы 2-4 не начинаются, пока базовый импорт не будет воспроизводимо запускаться.
