# Архитектура движка

Этап 2.5 реализует фазовый headless-движок, который может запускать партии без UI и готовит стабильный action/effect слой для будущих агентов.

## Источник данных

- Основной источник: `data/processed/cards_full.xlsx`.
- Нормализованный JSON: `data/processed/cards_full.json`.
- Загрузчик: `src.importers.xlsx_cards`.
- Игровая база: `src.game.data.load_card_database`.

Данные из таблицы не исправляются по догадке. Текст карты сохраняется как источник истины, а реализация эффекта помечается как `partial` или `not_implemented`.

## Основные модули

- `src.game.models` - dataclass-модели карт, игроков, состояния игры, действий и конфига.
- `src.game.setup` - подготовка партии, стартовые колоды, добор, рынок, стопки легенд/фамильяров.
- `src.game.engine` - применение действий, ход, покупка, победа над легендой, конец хода и проверка конца игры.
- `src.game.legal_actions` - единый генератор легальных действий.
- `src.game.targeting` - selectors: self, chosen enemy, all enemies, left/right, weakest/strongest, stronger/weaker.
- `src.game.deck_manifest` - загрузка и валидация физических тиражей стопок.
- `src.game.instances` - физические копии карт `CardInstance`.
- `src.game.card_text` - разделение текста карт на секции.
- `src.game.effects` - primitive effects и resolver. Сейчас реализованы `GainPower`, `DrawCards`, `Heal`, `DealDamage`, `DiscardCards`, `GiveWeakWand`, `RevealHand` и shell-примитивы для сложных эффектов.
- `src.game.replay` - запись debug replay.
- `src.game.replay_summary` - CLI-сводка по replay JSON для быстрой диагностики симуляций.
- `src.game.triggers` - shell для trigger-событий постоянок.
- `src.game.effect_coverage` - отчет покрытия реализации эффектов.
- `src.game.scoring` - подсчет победных очков и tie-breaker.
- `src.game.serialization` - JSON-сериализация состояния.
- `src.agents.random_agent` - RandomAgent для автосимуляций.

## Игровой цикл

1. `MAIN`: активный игрок разыгрывает карты, покупает карты или завершает ход.
2. `CHOOSE_TARGET`: создается, если эффект требует выбора цели.
3. `DEFENSE_WINDOW`: создается перед применением атаки, если у цели есть защита.
4. `RESOLVING_EFFECT`: автоматическое разрешение primitive effects.
5. `END_OF_TURN`: сброс руки и сыгранных непостоянных карт, добор новой руки.
6. `LEGEND_REVEAL`: раскрытие новой легенды и попытка разрешить групповую атаку.
7. `GAME_OVER`: финальное состояние и подсчет победителей.

## Реализовано

- 2-5 игроков.
- Стартовая колода: 6 знаков, 1 палочка, 3 пшика.
- Стартовое здоровье 20.
- Добор 5 карт и перемешивание сброса в колоду.
- Основная колода, барахолка на 5 карт, беспределы как reveal-placeholder.
- Отдельные стопки легенд, шальной магии, вялых палочек и фамильяров.
- Покупка карт в сброс.
- Победа максимум над одной легендой за ход.
- Централизованный legal action generation.
- Физические копии карт в игровых зонах.
- Manifest-тиражи для основной колоды, стартовых карт, шальной магии, вялых палочек, легенд и дохлых колдунов.
- Вум как первая легенда; число легенд зависит от числа игроков.
- Фамильяр покупается в сброс и затем разыгрывается/защищает только как карта из руки.
- Секционный парсинг текста: обычный розыгрыш не применяет защиту, защита не применяет main text.
- Pending target choice.
- Defense window: отказ, обычная защита, базовый добор/лечение из текста защиты, перенаправление урона фамильяром.
- Постоянки остаются в `ongoing` и не сбрасываются в конце хода.
- Смерть: игрок получает жетон дохлого колдуна и восстанавливается до временной константы `death_reset_health`.
- Условия конца: пустая основная колода при пополнении рынка, пустая стопка легенд, лимит дохлых колдунов, max-turns.
- Подсчет VP по зонам игрока, штрафы дохлых колдунов, tie-breaker.

## Ограничения

- Большинство эффектов карт пока не реализовано полностью.
- Большинство уникальных текстов карт все еще `partial` или `not_implemented`.
- Сложные защиты и атаки работают только как базовые парсируемые фрагменты.
- Групповые атаки легенд применяются только если из текста извлекается простой `нанеси N урон`.
- Групповые атаки легенд идут по всем колдунам в порядке от следующего игрока по кругу.
- Беспределы при выходе из основной колоды отправляются в resolver/log, но их индивидуальные эффекты в основном не реализованы.
- Свойства загружены, но еще не встроены в игровой цикл.
- Для основной колоды manifest пока фиксирует по 1 копии каждой уникальной основной карты/беспредела, потому что официального тиража в таблице нет.
